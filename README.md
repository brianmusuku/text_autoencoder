# HyperNetwork Text Autoencoder

A PyTorch implementation of a Text Autoencoder that utilizes a **HyperNetwork** decoder to reconstruct sequences from a compressed latent space. This project demonstrates how HyperNetworks can be used to generate weights for a position-based classifier, allowing for effective text reconstruction.

## 🚀 Overview

Traditional autoencoders use a fixed decoder architecture. In this implementation, the decoder's weights are dynamically generated by a **HyperNetwork** based on the latent representation produced by the encoder. This allows the model to "specialize" its decoding logic for each specific input.

### Key Features
- **Encoder**: Bidirectional LSTM that compresses variable-length text into a fixed-size latent vector.
- **HyperNetwork Decoder**: A neural network that takes the latent vector and generates weights and biases for an implicit representation classifier.
- **Positional Encoding**: Uses sinusoidal positional encodings to help the decoder identify the sequence position it is currently reconstructing.
- **Latent Space Analysis**: Built-in evaluation to check if the latent space naturally clusters similar topics (e.g., activation functions vs. loss functions).

## 📂 Repository Structure

- `model.py`: The core implementation containing the model architecture, training loop, and evaluation logic.
- `data.csv`: A dataset of 47 statements about neural network components (activations and losses), categorized by topic.
- `output_log.txt`: Example output showing training progress and evaluation results.

## 📊 Results

### Reconstruction
The model achieves near-perfect reconstruction of the training sequences. Example:
- **Target**: `the relu ( rectified linear unit ) is an activation function...`
- **Decoded**: `the relu ( rectified linear unit ) is an activation function...`

### Latent Space Clustering
The model demonstrates that even without explicit labels during training, the latent space naturally groups semantically similar sentences:
- **Same Topic Pair** (e.g., Activation vs. Activation): High Cosine Similarity (~0.38)
- **Different Topic Pair** (e.g., Activation vs. Loss): Low/Negative Cosine Similarity (~-0.16)

## 🧠 How it Works

1. **Tokenization**: Text is lowercased and split into tokens (words and punctuation).
2. **Encoding**: A Bi-LSTM processes the tokens, and the final hidden states are concatenated and projected into a `latent_dim` (default: 64).
3. **Weight Generation**: The HyperNetwork (Linear layers) takes the latent vector `z` and generates a weight matrix `W1` and bias `b1`.
4. **Implicit Decoding**: For each position in the sequence, the pre-computed positional encoding is multiplied by the generated `W1` and added to `b1`, then projected to the vocabulary size to produce logits.
5. **Loss**: Cross-entropy loss is used to compare the predicted logits with the original input tokens.
